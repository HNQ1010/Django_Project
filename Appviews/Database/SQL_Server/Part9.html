SQL Server Date Functions

    1) Returning the current date and time

      Function Description
      CURRENT_TIMESTAMP Returns the current sys date and time without the time zone part.
      GETUTCDATE Returns a date of a date as an integer number.
      GETDATE Return the current sys date and time of the operating sys on which the SQL server is running.
      SYSDATETIME Returns the current sys date and time with more fractional seconds precision than the GETDATE() function.
      SYSUTCDATETIME Returns the current sys date and time in UTC time.
      SYSDATETIMEOFFSET Returns the current sys date and time with the time zone.

      * CURRENT_TIMESTAMP()

      * GETUTCDATE()

      • The GETUTCDATE() function returns the current UTC time.
      • The GETUTCDATE() function calculates this value from the operating sys on which SQL Server is running.

      * GETDATE()

      • The GETDATE() function returns the current system timestamp as a DATETIME value without the database time zone offset.
      • The DATETIME value is derived from the Operating System (OS) of the server on which the instance of SQL Server is running.

      * SYSDATETIME()

      • The SYSDATETIME() function returns a value of DATETIME2 that represents the current system date and time
        of the server on which the SQL Server instance is running.
      • Note that the SYSDATETIME() function has more fractional seconds precision than the GETDATE() function.
      • The SYSDATETIME() is a nondeterministic function, therefore, views and columns
        that have expressions reference this function cannot be indexed.

      * SYSUTCDATETIME()

      • The SYSDATETIME() function returns a value of DATETIME2 that represents the current system date and time
        of the server on which the SQL Server instance is running.The datetime is in Coordinated Universal Time(UTC).
      • Notice that the SYSUTCDATETIME() function has more fractional seconds precision than the GETUTCDATE() function.

      * SYSDATETIMEOFFSET()

      • The SYSDATETIMEOFFSET() function returns a value of DATETIMEOFFSET(7) that represents the current system date and time,
        which also includes the time zone, of the computer on which the SQL Server instance is running.
      • You can also use the FORMAT() function to return the time zone offset as a string by.

    2) Returning the date and time Parts

      Function Description
      DATENAME Returns a date part of a date as a character string.
      DATEPART Returns a date part of a date as an integer number.
      DAY Returns the day of a specified date as an integer.
      MONTH Returns the month of a specified date as an integer.
      YEAR Returns the year of a specified date as an integer.

      * DATENAME()

      • The DATENAME() function returns a string, NVARCHAR type, that represents a specified date part of a specified date.
        • date_part : is a part of the date that you want to return.
        • input_date : is a literal date or an expression that can resolve to a time, date.
        date_part abbreviation
        year yy, yyyy
        quarter qq, q
        month mm, m
        dayofyear dy, y
        day dd, d
        week wk, ww
        weekday dw
        hour hh
        minute mi, n
        second ss, s
        milisecond ms
        microsecond mcs
        nanosecond ns
        TZoffset tz
        ISO_WEEK isowk, isoww

      • Note that DATENAME() is similar to the DATEPART() except for the return type.
        The DATENAME() function returns the date part as a character string whereas the DATEPART() returns the date part as an integer.

      * DATEPART()

      • Returns an integer which is a part of a date such as day, month, year.
      • The rest, similar to DATENAME() funcion.

      * DAY()

      • Returns integer value which represents the day of the month(1-31) of a specified date.
      • Syntax DAY(input_date), and it similar as DATEPART(day, input_date)
      • The DAY() function will return 1 if input value has only time part.

      * MONTH()

      • Similar to DAY() function.

      * YEAR()

      • Similar to DAY() and MONTH() function.
      • With just time part, it will return 1900.

    3) Returning a difference between two dates

    • DATEDIFF: Returns a difference in date part between two dates.
      • date_part : is the part of date that you want to compare between start_date and end_date.
        See valid date in returning date and time parts.
      • start_date and end_date : are the dates to be compare and in these type: DATE, DATETIME, DATETIMEOFFSET, DATETIME2, SMALLDATETIME or TIME.

    4) Modifying dates

      Function Description
      DATEADD Adds a value to a date part of a date and return the new date value.
      EOMONTH Returns the last day of the month containing the specified date, with an optional offset.
      SWITCHOFFSET Changes the time zone offset of a DATETIMEOFFSET value and preserves the UTC value.
      TODATETIMEOFFSET Transforms a DATETIME2 value into a DATETIMEOFFSET value.

      * DATEADD()

      • The DATEADD() function adds a number to a specified date part of an input date and returns the modified value.
        • date_part : is the part of date to which the DATEADD() function will add the value.
        • value : is an INTEGER number to be added to the date_part of the input_date.
          If the value is a decimal or float, DATEADD() will truncate the decimal fraction part. It will not round the number.
        • input_date : is a literal date value or an expression which can resolve to valid type: DATE, DATETIME, DATETIMEOFFSET, DATETIME2, SMALLDATETIME, TIME.
      • If you add a number of month to a date and the day of the date result does not exist, the DATEADD() will return the last day of the returned month.

      * EOMONTH()

      • The EOMONTH() function returns the last day of the month of a specified date, with an optional offset.
        • start_date : is a date expression that evaluates to a date.
        • offset : (optional) is an integer that specifies the number of months to add to the start_date.

      * SWITCHOFFSET()

      • The SWITCHOFFSET() function returns a DATETIMEOFFSET changed from the stored time zone offset to a new time zone offset.
        • expression : is an expression that can resolve to a DATETIMEOFFSET value.
        • time_zone : can be a character string in the format {+|-}TZH:TZM, or a signed integer of minutes.(E.g: +08:00, -07:00, or 120).

      * TODATETIMEOFFSET()

      • The TODATETIMEOFFSET() translates a DATETIME2 value into a DATETIMEOFFSET value.
        • expression : is an expression that can resolve to a DATETIME2 value.
        • time_zone : can be a integer or string in form {+|-}TZH:TZM. The range of {+|-}TZH:TZM between +14:00 to -14:00.

    5) Constructing date and time from their parts

      Function Description
      DATEFROMPARTS Return a DATE value from the year, month, day.
      DATETIME2FROMPARTS Returns DATETIME2 value from the date and time arguments.
      DATETIMEOFFSETFROMPARTS Returns a DATETIMEOFFSET value from the date and time arguments.
      TIMEFROMPARTS Returns a TIME value from the time parts with the precisions.

      * DATEFROMPART()

      • The DATEFROMPART returns a date from year, month and day values.
      • If any argument is NULL, the function will return NULL
      • IF any argument is out of range, the function will raise an error.

      * DATETIME2FROMPART()

      • The DATEFTIME2ROMPART returns a DATETIME2 from year, month, day, hour, minute, second, fraction and precision values.
      • If any argument is NULL, the function will return NULL
      • IF any argument is out of range, the function will raise an error.

      * DATETIMEOFFSETFROMPART()

      • The DATEFTIMEOFFSETROMPART returns a DATETIMEOFFSET from specified date and time arguments.
      • If any argument is NULL, the function will return NULL
      • IF any argument is out of range, the function will raise an error.

      * TIMEFROMPARTS()

      • The TIMEFROMPARTS() function returns a fully initialized time value. It requires five arguments.
      • If any argument is NULL, the function will return NULL
      • IF any argument is out of range, the function will raise an error.

    6) Validating date and time values

    • ISDATE : Check if a value is a valid date, time or datetime value
    • The ISDATE() function accepts an argument and returns 1 if that argument is a valid DATE, TIME, or DATETIME value; otherwise, it returns 0.
    • ISDATE returns 0 if the expression is a datetime2 value.
    • Note that the range for datetime data is 1753-01-01 through 9999-12-31, while the range for date data is 0001-01-01 through 9999-12-31.
    • ISDATE is deterministic only if you use it with the CONVERT function, if the CONVERT style parameter is specified, and style is not equal to 0, 100, 9, or 109.
    • The return value of ISDATE depends on the settings set by SET DATEFORMAT, SET LANGUAGE and Configure the default language Server Configuration Option.
