WMS IMPROVEMENT

    ★ Issues

    • At this time, as the business growing, we need 2 systems serving our operation:

        • ☛ OPS (Operating system): Serve the operation between departments.

        ☛ WMS (Warehouse management system): Control warehouse inventory and streamline warehouse activities.

    • In other word, WMS is proportion of OPS, but it is related to main part (company's product) and can raise many issues involving in all departments.
    • Based on that, WMS improvement is the urgent task that need to be interested in at this time.
      • Firstly, As the current warehouse operation flow, the warehouse activities was missing 2 functions of the flow on the system. They are:

          • ☛ Put away (store) process

          ☛ Handover process

      • The second: WMS functions were hidden in different functions and process directly on that functions.(Receiving is in Buyer function and processes directly in it).
      • The third: A big important thing is Barcode system is not showing on the system in warehouse operation activities. It lead to:

          • ☛ Missing confirmation between operation processes → loosen process.

          ☛ Many manual processes that lead to make mistake easily in operation (Picking, packing without confirming information on each item, just identify by seeing)

          ☛ Staff have to double check for each finished processes.(Re-check picking list by product's name :(( ) → Cause of workload.

    ★ Next steps

    • As discussion with team and your suggestion to pick some most important features that need to improve. I'd like to show you steps as belows:
      • Step 1: Assign PIC to contact when we are starting project. We need a person that deeply understand and keep following our needs to make corresponding between system and warehouse operation.
      • Step 2: Gathering warehouse management system as one function and separating each function as warehouse operation flow.
      • Step 3: Apply/ show product's barcode on the system.
      • Step 4: After finishing above steps, and as the priorities in built-in these functions/ features. I'd like to focus on improving the Outbound operation. The features are:

          • ☛ Picking:

          Current Process Expected Process
          https://drive.google.com/file/d/1aZjOJR29vIJs6NYLYhCYCBxZkhHavxV-/view?usp=sharing

          Disadvantages:

          • Picking process was combining with packing and handover processes.
          • Staff are picking items by seeing the item's name → easily get wrong or mistaken items.
          • There're item's information (scan to see right or wrong items → It's just a reference document and it's not forced to process) → We just contain the issues when we do the process relate to that item.

          Advantages:

          The expected picking process will:

          • Make confirmation in each item's barcode when staff do picking process. Make sure the items are picked correctly at right position.
          • Combine re-check process at ech item that was picked.
          • Make picking process more auto, quickly, and minimize actions that based on human observation that easily lead to mistakes or wrongs.
          ⚑ Inside picking function: There're 3 columns (or tabs) that showing these information:

            ⚐ Pending: these orders with "Out of stock" Status or facing issues.

            ⚐ Ready to pick: 2 options:

            • B2C orders:Limit the amount of items in each picking list (150 - 200 items)
            • B2B orders:Manually choose which order need to pick in each picking list.

            ⚐ Picked/ Returning

            ★ Packed column should belong to Packing function and Requested column should belong to Handover function.

          ⚑ The creating picking list process will be as regular but the visualization in picking list will be displayed these information:

            ⚐ Input place to scan item's barcode for confirming right item's barcode.

            ⚐ Item's position (where staff can pick item).

            ⚐ Input place for entering item's quantity for confirming after staff have picked items.

          • These step in Picking list is replacing for Re-check step, so we can ignore Re-check step and update inventory after staff confirm item's quantity.

          ☛ Packing:

          Current Process Expected Process
          https://drive.google.com/file/d/1KdmtorUqYmEIZ0nkzhrMS1v7IMopcC5a/view?usp=sharing

          Disadvantages

          • B2C process: It's not divided volume into particular packing lists. It sum all of orders waiting for packing. The problems are:

              • ☛ Cannot control the packaging time in case the volume is increasing.

              ☛ Cannot do picking process while packaging orders.

              ☛ When there're more than 1 staff do packaging process, the visualization in packing page will be illogical and we can't
              contain the overview of the packing situation.

              ☛ Beside, the updates of data of picking, packing while staff are doing one of these process can raise system errors or
              bugs in operation system.

          • B2B process

              • ☛ Manually do case by case for each step inside process.(status, picking, packing, handover, ...) → manually process.

              ☛ Separate from packing process. (Process in another function (Distribution function) and directly process in it).

              ☛ Cannot control the order's details exactly (there're not confirming of picking, packing, handover process on the system).

          Advantages:

          The expected packing process will:

          • Make confirmation when staff do packaging process on each item's barcode and item's quantity.
          • Separate each packing list for each staff at the same time to get the overview of packing situation.
          • Make process more automatic, quickly, minimize actions that based on human observation that easily lead to mistakes or wrongs.
          ⚑Inside Packing function: There're 3 options:

            B2C

            ☛ Waiting for packing list: It's corresponding with B2C picking list that was picked and visualizing these information:

            • Order code
            • From order code showing direct to item's barcode (with product's picture and product's name) and quantity requested.

              • ☛ when staff press "Packing" button, the system will present one by one order with:

              • Input places for item's barcode confirming
              • Input places for item's quantity confirming
              • Print button to print out order label

            B2B

            ☛Waiting for packing list: Corresponding with B2B picking list that was picked and visualizing these information:

            • Order code
            • From order code showing direct to item's barcode (with product's picture and product's name) and quantity requested.

              • ☛ when staff press "Packing" button, the system will present the first BOX with:

              • Input places for item's barcode confirming
              • Input places for item's quantity

                • ☛ system will auto minus the item's quantity on each BOX until the last BOX.

                At the last BOX, there will be "Print label" button to print out all of BOX label (Eg: 1/10, 2/10, ...) of the order.

          Packed order

          • Similar with Requested page inside Packing function at this time. It includes orders that was packed and waiting for handover process.
