BAR

  • In a bar plot, each row of data_frame is represented as a rectangular mark.

    • BAR with Plotly Express Figure

    Attributes list:

    • data_frame(DataFrame or array-like or dict) :This argument needs to be passed for column names (and not keyword names)
      to be used. Array-like and dict are transform internally to a pd.DataFrame . If missing, a DataFrame gets constructed under
      the hood using the other arguments.
    • x (str, int, Series, array-like) : Values from this column or array-like are used to position marks along the x-axis in
      Cartesian coordinates.
    • y (str, int, Series or array-like) :Value from this column or array-like are used to position marks along the y-axis in
      Cartesian coordinates.
    • color(str, int, Series or array-like) :Values from this column or array-like are used to assign color to marks.
    • pattern_shape(str, int, Series or array-like) :Value from this column or array-like are used to assign pattern shapes to marks.
    • facet_row(str, int, Series or array-like) :Values from this column or array-like are used to assign marks to faceted subplots
      in the vertical direction.
    • facet_col(str, int, Series or array-like) :Value from this column or array-like are used to assign marks to faceted subplots in
      the horizontal direction.
    • facet_col_wrap(int) :Specify number of facet columns per row. Ignored if 0, and forced to 0 if facet_row or marginal is set.
    • facet_row_spacing(float between 0 and 1) :Spacing between facet rows, in paper units. Default 0.03 or 0.0.7 when facet_col_wrap is used.
    • facet_col_spacing(float between 0 and 1) :Spacing between facet col, in paper units. Default 0.02
    • hover_name(str, int, Series or array-like) :Values from this col or array-like appear in bold in the hover tooltip.
    • hover_data(list of str, int, Series, array-like or dict) :With dict, column names as keys, value is True(appear) of False(disappear).
      Values from these appear as extra daa in the hover tooltip.
    • custom_data(list of str, int, Series or array-like) :Values from these columns are extra data, to be used in widgets or Dash callback.
      This data is not user-visible but is included in events emitted by the figure.
    • text(str, int, Series or array-like) :Values from this appear in teh figure as text labels.
    • base(str, int, Series or array-like) :Values from this is used to position the base of the bar.
    • error_x(str, int, Series or array-like) :Values from this is used to size x-axis error bar. If error_x_minus is None, error bar
      will be symmetrical, otherwise error_x is used for the positive direction only.
    • error_x_minus(str, int, Series or array-like) :Values from this is used to size e-axis error bars in teh negative direction.
      Ignored if error_x is None.
    • error_y(str, int, Series or array-like) :Values from this is used to size y-axis error bars. If error_y_minus is None, error bar
      will be symmetrical, otherwise error_y is used for positive direction only.
    • error_y_minus(str, int, Series or array-like) :Values from this is used to size y-axis error bar in the negative direction.
      Ignored if error_y is None.
    • animation_frame(str, int, Series or array-like) :Values from this is used to assign marks to animation frames.
    • animation_group(str, int, Series or array-like) :Values from this is used to provide object-constancy across animation frames: rows with
      matching animation_group will be treated as if they describe the same object in each frame.
    • category_order(dict with str keys and list of str value) :Default {}. This parameter is used to force a specific ordering of values
      per column. The keys of this dict should correspond to column names, and the values should be lists of strings corresponding to the
      specific display order desired.
    • labels(dict with str key and str value) :Default {}. By default, column names are used in the figure for axis titles, legend
      entries and hovers. This parameter allows this to be overridden. The keys of this dict should correspond to column names, and the values
      should correspond to the desired label to be displayed.
    • color_discrete_sequence(list of str) :String should define valid CSS-colors. When color is set and the values in the corresponding column are
      not numeric, values in that column are assigned colors by cycling through color_discrete_sequence in teh order described in category_orders,
      unless the value of color is a key in color_discrete_map. Various useful color sequences are available in the plotly.express.colors(plotly.express.colors.qualitative).
    • color_discrete_map(dict with str keys and str values) :Default {}. String values should define valid CSS-colors.Used to override color_discrete_sequence
      to assign a specific colors to marks corresponding with specific values. Keys in color_discrete_map should be values in the column denoted by color.
      Alternatively, if the values of color are valid colors, the string 'identity' may be passed to cause them to be used directly.
    • color_continuous_scale(list of str) :Strings should define valid CSS-colors. This list is used to build a continuous color scale when
      the column denoted by color contains numeric data. Various useful color scales are available in the plotly.express.colors (plotly.express.colors.sequentials,
      plotly.express.colors.diverging and plotly.express.colors.cyclical).
    • pattern_shape_sequence(list of str) :String should define valid plotly.js patterns-shapes. When pattern_shape is set, values in this are assigned patterns-shapes
      by cycling through pattern_shape_sequence in the order described in category_orders, unless the value of pattern_shape is a key in pattern_shape_map.
    • pattern_shape_map(dict with str keys and str values) :Default {}. Strings values define plotly.js patterns-shapes. Used to override
      pattern_shape_sequence to assign a specific patterns-shapes to lines corresponding with specific values. Keys in pattern_shape_map should be values
      in the column denoted by pattern_shape. Alternatively, if the values of pattern_shape are valid patterns-shapes names, the string 'identity' may be passed
      to cause them to be used directly.
    • range_color(list of 2 numbers) :If provided, override color_continuous_Scale.
    • opacity(float) :Value between 0 and 1. Sets the opacity for markers.
    • orientation('v' or 'h') :Default 'v' if x and y are provided, continuous or category. Otherwise, 'v'('h') if x(y) is categorical and y(x) is continuous.
      'v'('h') if x(y) is provided.
    • barmode('group', 'overlay', 'relative') :Default 'relative', bars are stacked above zero for positive values and below zero for negative values.
      In 'overlay', bars are drawn on top of one another, and 'group' mode, bars are placed beside each other.
    • log_x(boolean) :Default False. If True, the x-axis is log-scaled in cartesian coordinates.
    • log_y(boolean) :Default False. If True, the y-axis is log-scaled in cartesian coordinates.
    • range_x(list of 2 numbers) :If provided, override auto-scaling on the x-axis in cartesian coordinates.
    • range_y(list of 2 numbers) :If provided, override auto-scaling on the y-axis in cartesian coordinates.
    • title(str) :The figure title.
    • template(str or dict or go.layout.Template instance) :The figure template name(must be a key in plotly.io.templates) or definition.
    • width(int) :Default None. The figure width in pixels.
    • height(int) :Default None. The figure height in pixels.

    BAR Charts with Plotly Express

  • Plotly express is easy-to-use, high level interface to Plotly.
  • With px.bar, each row of the DataFrame is represented as a rectangular mark.
    •             import plotly.express as px
            data_canada = px.data.gapminder().query("country == 'Canada'")
            fig = px.bar(data_canada, x = 'year', y = 'pop')
            fig.show()

    BAR Charts with Long Format Data

  • Long-form data has one row per observation, and one column per variable. This is suitable for storing and displaying multivariate data
    with dimension greater than 2. This format is sometimes called 'tidy'.
    •             df = px.data.medals_long()
            fig = px.bar(df, x = 'nation', y = 'count', color = 'medal', title = 'Long-Form Input')
            fig.show()

    BAR Charts with Wide Format Data

  • Wide-format data has one row per value of one of the first variable, and one column per value of second variable. This is suitable
    for storing and displaying 2-dimensional data.
    •             df = px.data.medals_wide()
            fig = px.bar(df, x = 'nation', y = ['gold', 'silver', 'bronze'], title = 'Wide-form Input')
            fig.show()

    BAR Charts in Dash

                import dash
            from dash import dcc
            from dash import html
            from dash import Input, Output
            df = px.data.tips()
            days = df.day.unique()
            app = dash.Dash()
            app.layout = html.Div([dcc.Dropdown(id = 'dropdown', options = [{'label': x, 'value':x} for x in days],
                                    value = days[0], clearable = False]), dcc.Graph(id = 'bar-chart')])
            @app.callback(Output['bar-chart', 'figure'], [Input('dropdown', 'value')])
            def update_bar_chart(day):
                mask = (df['day'] == day)
                fig = px.bar(df[mask], x = 'sex', y = 'total_bill', color = 'smoker', barmode = 'group')
                return fig
            app.run_server(debug = True)

    Customize Bar Charts with Plotly Express

  • The bar plot can be customized using keyword argument (color, color_line, ...)
    •             df = px.data.gapminder().query("country == 'Canada'")
            fig = px.bar(df, x = 'year', y = 'pop', hover_data = ['lifeExp', 'gdpPercap'], color = 'lifeExp',
                        labels = {'pop' : 'Population of Canada'})
            fig.show()
  • When several rows share the same value of x (Male and Female), the rectangles are stacked on top of of another.
    •             df = px.data.tips()
            fig = px.bar(df, x = 'sex', y = 'total_bill', color = 'time')
            fig.show()
  • The default stacked bar chart behavior can be changed to grouped (also known as clustered) using the barmode argument.
    •             fig = px.bar(df, x = 'sex', y ='total_bill', color = 'time', barmode = 'group')
            fig.show()
  • Bar chart also support using pattern.(pattern_shape, pattern_shape_sequence, pattern_shape_map, ...)
    •             df = px.data.medals_long()
            fig = px.bar(df, x = 'medal', y = 'count', color = 'nation', pattern_shape = 'nation',
                            pattern_shape_sequence = ['+', 'x', '.'])
            fig.show()

    Faceted Subplot

  • Use the keyword argument facet_row( or facet_col) to create faceted subplots, where different rows(columns) correspond to
    different values of the DataFrame column specified in facet_row. 
    •             df = px.data.tips()
            fig = px.bar(df, x = 'sex', y = 'total_bill', color = 'smoker', barmode = 'group',
                            facet_row = 'time', facet_col = 'day', category_order = {'day' : 'Thur', 'Fri', 'Sat', 'Sun',
                                                                                    'time' : 'Lunch', 'Dinner'})
            fig.show()

    Controlling Text Fontsize with uniformtext

  • If you want all the text labels to have the same size, you can use the uniformtext layout parameter. The minsize attribute sets
    the font size, and the mode attribute sets what happens for labels which cannot fit with the desired fontsize: either hide them or
    show them with overflow.
  • texttemplate : Template string use for rendering the information text that appear on points. It will override 'textinfor'.
    Variables are value using for : label, color, value, percent and text. Variables are inserted using %{variable}.

    Bar Chart with Relative Barmode

  • With 'relative' (stack) barmode, the bars are stacked on top of one another, with negative values below, positive above the axis.
    •             x = [1, 2, 3, 4]
            fig = go.Figure()
            fig.add_trace(go.Bar(x = x, y = [1, 4, 9, 16])
            fig.add_trace(go.Bar(x = x, y = [6, -8, -4.5, 8])
            fig.add_trace(go.Bar(x = x, y = [-15, -3, 4.5, -8])
            fig.add_trace(go.Bar(x = x, y = [-1, 3, -3, -4])
            fig.update_layout(barmode = 'relative', title_text = 'Relative Bar Mode')
            fig.show()
